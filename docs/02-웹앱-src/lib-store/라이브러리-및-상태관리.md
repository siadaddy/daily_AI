# 라이브러리 및 상태관리

`src/lib/`과 `src/store/`의 작은 공용 모듈들을 하나로 묶는다. 각 소제목이 하나의 파일(또는
밀접한 파일 묶음)에 대응한다.

#### supabase/client.ts, server.ts, admin.ts
세 개의 Supabase 클라이언트 팩토리, **용도가 겹치지 않게 엄격히 분리**되어 있다.

| 파일 | 함수 | 사용처 |
|---|---|---|
| `client.ts` | `createBrowserClient` | Client Component 전용 |
| `server.ts` | `createServerClient` (쿠키 기반 세션) | Server Component / Route Handler / Server Action 전용 |
| `admin.ts` | `createAdminClient` (service-role) | RLS를 우회하므로 리포트 upsert 등 서버 전용 코드에서만. `'use client'` 파일에서 절대 import 금지 |

`admin.ts`는 `SUPABASE_SERVICE_ROLE_KEY`가 없으면 즉시 throw한다 — 조용히 실패하지 않도록.

#### reports/generate.ts
[[reports-generate]], [[analytics]]가 공유하는 리포트 생성 로직 + `fetchAllPages` 페이지네이션
헬퍼(Supabase 1000행 제한 우회). 자세한 내용은 [[reports-generate]] 참고.

#### hooks/useChartColors.ts
`'use client'` 훅. CSS 커스텀 프로퍼티(`--chart-grid`, `--viz-1`~`--viz-6` 등)에서 색을 읽어
Chart.js에 넘긴다. `MutationObserver`로 `data-theme` 속성 변화를 감지해 다크/라이트 전환 시
자동 재계산 — Chart.js 색상을 하드코딩하지 말고 반드시 이 훅을 통해서 가져올 것
([[reports]]의 4개 차트가 사용).

#### hooks/useInterval.ts
`setInterval`을 선언적으로 쓰기 위한 표준 React 패턴(`useRef`로 최신 콜백 유지). `ClockWidget`
등에서 사용.

#### utils/keywords.ts
한국어 키워드 추출. 조사/어미 등 불용어(`STOPWORDS`) 목록을 갖고 있다. [[analytics]],
[[reports-generate]]가 제목/트렌드/본문에서 상위 키워드를 뽑는 데 사용.

#### utils/caption.ts
- `highlightCaption()` — `[사실]`(초록) `[분석]`(파랑) `[전망]`(보라) 태그를 색상 강조 HTML로
  변환. [[newsletter]]의 `NewsCard`가 사용.
- `mdToHtml()` — 마크다운(`#`/`##`/`###`/`-`/`**bold**`/`>`/해시태그 줄)을 HTML로 변환.
  [[newsletter]]의 `BlogArticle`이 [[articles]] 본문 렌더링에 사용.

#### utils/validation.ts
Server Action(주로 [[community-테이블]] 관련)에서 사용자 입력을 검증하는 공용 헬퍼.
- `LIMITS` — `postTitle: 100`, `postContent: 5000`, `comment: 1000`, `nickname: 20`.
- `validateText(raw, label, maxLength)` — trim 후 빈 값/길이 검증, 실패 시 한국어 에러 메시지.
- `isValidContentKey(key)` — `content_key` 형식 검증(영문/숫자/`-_:.`, 최대 200자).
- `publicDbError(error, context)` — Supabase 원본 에러 메시지를 사용자에게 노출하지 않고
  서버 로그로만 남기며 일반화된 한국어 메시지를 반환.

새 사용자 생성 콘텐츠 기능(댓글, 게시글 등)을 추가할 때는 이 파일의 헬퍼를 그대로 재사용할 것
— 코드 주석에도 명시된 규칙.

#### types/index.ts
모든 Supabase 테이블 행 타입(`NewsCard`, `ContentCard`, `Article`, `Agent`, `ActivityLog`,
`NewsTrend`, `PeriodReport`/`WeeklyReport`, `CommunityPost`, `CommunityComment` 등)과 UI 타입
(`TabId`, `Category`)이 여기 모여 있다. 각 [[00-스키마-개요|Supabase 테이블 노트]]의 컬럼 표는
이 파일을 근거로 작성했다.

#### site-url.ts
`getSiteUrl()` — `NEXT_PUBLIC_SITE_URL` → Vercel 프로덕션 URL → `localhost:3000` 순으로 폴백.
OG 메타데이터 등 절대 URL이 필요한 곳에서 사용.

#### store/app.ts (Zustand)
`activeTab`, `categoryFilter`, `selectedDate` — [[00-src-개요|URL 쿼리 상태를 클라이언트에서
미러링]]하는 용도. `selectedDate`의 기본값은 KST 기준 오늘 날짜(`todayKST()`,
`Intl.DateTimeFormat`으로 타임존 계산, UTC 서버에서도 정확히 KST 날짜가 나오게 함).
